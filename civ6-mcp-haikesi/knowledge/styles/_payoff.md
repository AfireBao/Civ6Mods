# 收益优先选卡策略（payoff）

仅当本轮掷骰为**收益优先**（或无风格标签）时注入。  
**cosplay 模式不使用本篇**——那时只跟风格 Skill + 合法性底线。

目标：在合法候选内最大化本文明可兑现收益与战局存活，用工具核对后再选。

## 工具

- 在建 / 人口 / 宜居 → `city_pressure`
- 贴脸敌军 → `border_threats`
- 兵种维护费、战斗力、解锁科技/市政 → `civilopedia_lookup`
- 宜居规则摘要 → `civ6_kb(amenity)`；商路 → `civ6_kb(trade)`

## 即时 vs 延迟（AI 海克斯标签）

- 优先**【即时】**或**【条件即时】**且条件已满足的牌。
- **【延迟·需能生产 X】**：`city_pressure` 已有同系在建 → 抬权；**无同系在建 ≠ 禁止**。
  若科技/市政已解锁（或约数回合内可解锁）且局势需要该类单位，仍可选——注明将改队列兑现，视为延迟收益而非空放。
  仅当该兵种长期造不出（缺前置、远古早期军力极弱却押工业重骑等）才当近似空放、强烈降权。
- **石弩 / 攻城器 = 攻城 ≠ 远程**；近战 / 骑兵 / 远程 / 攻城分系对齐。错系 echo（把投石机当远程）→ 不选。

## 0 城与资源创建

- **0 城禁止**资源创建（奶龙/丝绸/烟草/茶等）——效果空放。
- 有城：创建落在**最新城 3 环**；看地块产出 + 改良后宜居。

## 奢侈与宜居

- 每种**不同**奢侈改良后，最多为 4 城各 +1 宜居；同种多余拷贝不加宜居种类。
- 宜居盈余可带来产出加成；赤字惩罚产出/忠诚——`city_pressure` 有赤字时抬权新奢侈类型或宜居向创建。
- 资源创建文案中的「地块+食物/科技/文化…」是原版奢侈固有收益，计入即时价值。

## 科技 / 市政与战斗优势

- 即时**科技%** / 能加快军事解锁的牌：在交战或即将换代兵种时抬权。
- **领先科技与市政**可解锁更强单位、政策与政体——不必只盯当前在建；`civilopedia_lookup` 确认兵种 PrereqTech / 维护费后再押科技%。
- 文化% 对政策卡与部分征服辅助仍有价值，按 RST 权衡。

## 金币与维护费

- 多数军事单位有**每回合维护费**；具体 Cost / Maintenance 查词典，勿臆造。
- 金币赤字或大军在养：抬权即时**金币%**；盲目 echo 爆兵可能拖垮经济。
- 交战弱势：先保证能造、养得起的单位，再谈翻倍。

## 商路互利

- 两河 / 天朝等「需他国商路打入」：无国际入向或交战中 → 强烈降权（常空放）。

## 混乱干扰（收益视角）

- 南蛮 / 仇水等：评估是否拖慢**对手**多于拖慢己方（触发者常免疫）；己方已满战争/蛮族压境时可降权。
- 仇水连汛：先 `flood_targets`；可见目标可泛滥河合计=0 → 强烈降权（空放）。
- 不因「听起来猛」而忽视即时%或产线对齐的 echo。

## 百分比产出

- 全城市科技/文化/金币/信仰/食物/生产力%：基盘越高收益越大；单城微弱时资源创建或 echo 可能更好。

## 知识边界

- 海克斯效果以本轮候选全文为准；历史库存才 `lookup_relic`。
- 专名数值走字典，不把过时数字写进策略长期维护。
